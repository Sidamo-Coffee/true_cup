# true_cup
## サービス概要
難しい知識がなくても、自分にぴったりのコーヒーが見つかります。
味覚診断と飲んだコーヒーの記録から、あなたの好みをわかりやすく可視化。
「自分の基準」で豆やお店を選べるようになるためのサポートアプリです。

## このサービスへの思い・作りたい理由
これまで「コーヒーは好きだけど、自分の好みがよく分からない」という方々に多く出会ってきました。

私自身も、以前は「なんとなく」でお店やメニューを選んでいました。しかし、多くのお店を訪れ数多くコーヒーを飲んでいるうちに、「好きな味や香り」が少しずつ分かるようになり、自分好みの基準でお店や豆を選べるようになりました。

一人でも多くの方により手軽に、楽しみながら好みの味を見つけて、コーヒーの奥深さ・面白さを知ってもらいたい。

そんな思いから生まれたサービスです。

## ユーザー層について
| 層 | 理由 |
| ------------------------------ | --------------------------------------------- |
| 自分がどんな味を好むのかを簡単に把握したいライトユーザー | 詳しい知識はいらないが、「自分に合う一杯」は知りたいという需要が多い |
| カフェ巡り・喫茶店巡りが好きで、新しいお店を開拓したい人 | 自分の味覚傾向を元にお店を選ぶ体験が、より満足度の高いものになる |
| コーヒーの世界に興味はあるが、専門的な学習はハードルが高い人 | 本やインターネットで調べると専門的すぎて逆に混乱してしまうことがあるので、シンプルなイラストや説明で学んでほしい |


## サービスの利用イメージ

このアプリは、コーヒーに詳しくなくても、自分の好みの傾向が自然にわかるように設計されています。
最初に簡単な味覚診断を受け、あとは飲んだコーヒーを気軽に記録していくだけで、あなたの「好きな味」「よく選ぶ傾向」「次に試すべき一杯」が見えてきます。

カフェ巡りが好きな方は、新しいお店の発見に役立ちます。
ライトユーザーは、「自分の好き」が少しずつ分かる楽しさを感じられます。
難しい知識を学ばなくても、自分の基準でコーヒーを選べるようになる。
そんな体験を目指しています。

### サービスの利用手順

1. 味覚タイプ診断を受ける  
   甘味・酸味・苦味・コクなど、数問の質問に答えるだけで、あなたの味覚傾向が表示されます。

2. 飲んだコーヒーを簡単に記録する  
   写真や一言メモ、感じた特徴（酸味・苦味など）を選ぶだけで登録できます。

3. 自分の好みの傾向が可視化される  
   記録データから、よく選ぶ焙煎度・産地・味の傾向が自動でまとめられます。

4. おすすめの豆・お店が表示される  
   好みの傾向をもとに、おすすめのコーヒー豆や焙煎度が表示され、迷わず選べるようになります。

5. 記録を重ねるほど、自分の"好き"が深まる  
   いろんなコーヒーを飲むほど、味覚マップの精度が上がり、自分の基準が育っていきます。

## ユーザーの獲得について

1. 味覚診断機能 × SNS共有  
   診断結果を画像（カード）にしてSNSで共有。  
   (例)  
   「あなたは香ばし系・中深煎りタイプ」  
   「あなたのタイプは○○％が深煎り好き」

2. カフェや喫茶店を訪問して紹介&ヒアリング  
   お店の店主や常連客にオフラインでアプリを紹介。  
   機能やUIについてヒアリングしてフィードバックをもらう。

3. プログラミングスクール内コミュニティ or コーヒー系コミュニティで紹介  
   アプリ開発 or コーヒーに関心のある人の多い場所で発信。

## サービスの差別化ポイント・推しポイント
■ 初心者に特化した“やさしい”コーヒーアプリ

専門用語や知識がなくても使えるよう、感覚的かつシンプルな UI で設計。
本格派向けの記録アプリとは違い、コーヒーに詳しくない人でも継続しやすい点が特徴。

■ 味覚診断 × 記録 × 可視化 × おすすめ の一気通貫体験

最初の味覚診断で「自分の好みの仮説」をつくり、
飲んだコーヒーを記録するほど好みの傾向が明確に。
診断だけで終わらず、好みが育っていく“成長型”の体験が可能。

■ 自分の“味の基準”がわかる可視化機能

飲んだコーヒーを登録するだけで、
「どんな味をよく選んでいるか」「どの焙煎度が好きか」などが自動で整理。
自分では気づきにくい“味覚のクセ”や好みが見える化。

■ コーヒーの自己理解を深める新しい体験

「コーヒー記録アプリ」でも「診断アプリ」でもなく、
自分の味覚を知り、育て、選べるようになる ことに特化したアプリ。
飲むほどに自分の基準が磨かれ、新しい一杯との出会いが楽しくなるようサポート。

## 機能候補
### MVPリリース
- 味覚診断
- 記録機能
- 好みの傾向可視化機能(簡易版: よく選ぶ焙煎度・味の傾向がまとめられる程度)
- おすすめ表示機能(簡易版: おすすめ焙煎度のみ表示)
- シェア機能（味覚診断結果の共有、シンプルなテキストでの共有機能）
- 認証機能

### 本リリース
- 好みの傾向可視化機能の高度化
- おすすめ表示機能の高度化
- 店舗ページ(全ユーザー記録から集計された味の特徴表示)
- コーヒーミニ辞書機能


## 使用する技術スタック
| 種類 | 採用技術 | 目的・補足 |
| ----------- | --------------------------------------- | ---------------------------------------------------------------------- |
| **フロントエンド** | **HTML / CSS / Tailwind CSS** | Rails 7 と相性が良く、開発効率が高い。UIの統一とパーツ管理がしやすい。初心者向けにシンプルな画面を構築可能。 |
| | **Turbo / Stimulus(Hotwire)** | SPAほど複雑にせず、ページ遷移の高速化・リアクティブなUIを実現。味覚診断やフォーム周りと相性が良い。 |
| | **Chartkick** | 味覚傾向のグラフ表示(円グラフ・棒グラフ)に使用。可視化機能のMVPを最速で実装しやすい。 |
| **サーバーサイド** | **Ruby on Rails 7系** | MVC構造で実装しやすく、診断ロジックやおすすめ機能などをサービス層で整理しやすい。個人開発との相性が良い。 |
| **データベース** | **PostgreSQL** | Railsとの相性、JSON型サポート、将来の集計・分析に強い。店舗データや味覚ログの集計が扱いやすい。 |
| **デプロイ** | **Render** | 無料〜低コストでデプロイでき、Railsアプリ運用の学習にも向く。PostgreSQLアドオンも利用しやすい。Heroku代替として現実的。 |
| **認証** | **Devise** | ユーザー登録・ログイン・パスワード管理を担当。初期MVPで最も安定して使いやすい。 |
